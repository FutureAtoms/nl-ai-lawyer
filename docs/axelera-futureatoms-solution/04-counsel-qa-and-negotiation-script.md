## 3) Answers to Your 13 Counsel Questions

### Conflict of interest

1. Is "having a company" itself a conflict if no revenue?
- Usually no automatic conflict by itself. Risk depends on overlap, resources, confidentiality, and role duties. Use written approval + boundaries.

2. How to structure disclosure + approval?
- Signed side-business addendum with: approved activity scope, objective restrictions, time/resource separation, periodic disclosure, and escalation process.

3. How to handle "precedent" argument?
- Offer a policy-based model: approval criteria applicable to all employees, not a personal exception. Use objective tests (competition, confidentiality, duty hours).

4. Does title/role change help or hurt?
- Can help if role scope is explicitly defined and matched with conflict boundaries. It hurts only if role wording becomes broader while restrictions stay vague.

### IP ownership / inventions

5. Which clauses prevent future-invention capture?
- Add explicit carve-out: off-hours, own equipment, outside Axelera duties, no Axelera confidential info => FutureAtoms ownership.

6. How prevent hardware testing from automatic Axelera ownership?
- Contract language: test access does not transfer ownership in pre-existing or independent FutureAtoms IP; only Wingman deliverable output rights transfer.

7. How define Axelera confidential info so ChipOS stays clean?
- Include standard exclusions: public domain, prior lawful possession, independent development, third-party lawful receipt.

### Code delivery / versioning

8. Wording for one-version delivery only?
- "Supplier delivers Wingman v1.0.0 only. Updates, upgrades, new features, and future versions are excluded unless separately agreed in writing."

9. Best delivery mechanism?
- Tag + signed source archive + checksums + manifest + account transfer log. Optional escrow only if continuity concern remains.

10. Legal risk that Wingman is a fork?
- Manageable if provenance is documented and excluded ChipOS modules are clearly removed/listed. Clean-room is usually unnecessary if licensing boundary is explicit.

### Marketing / publicity

11. Language for marketing rights?
- Mutual prior written approval per publication; approval not unreasonably withheld/delayed; factual statements allowed after approval.

### Employment vs contracting

12. Contractor vs employee impact on residence/permanent residency/citizenship?
- Employee route is usually operationally safer while naturalisation planning is active. Contractor route should be used only after immigration specialist validates permit implications and timeline.

13. Can invoices be used while remaining employee?
- High tax/labor risk if invoiced scope overlaps employment duties (Wet DBA/schijnzelfstandigheid). Prefer employment bonus/equity for Wingman while employed.
